Component-Scan(basePackage):-

>> All your classes (programmer defined files) must be under
    Starter class/Main class package (or its subpackage)

>> if your create your class outside of Main class package (or any diff package)
   then Spring container can not find your clas, no object is created.

Example:
 Main class Package  : in.nareshit.raghu

 Product             : in.nareshit.raghu   (Detected)

 Admin               : in.nareshit.raghu.model  (Detected)

 Employee            : in.nareshit     (Not Detected)

 DatabaseCon         : com.app.model   (Not Detected)

 ExcelExport         : in.nareshit.raghu.a.b.c.d.e (Detected)

*) Spring Boot , by default gives Main class Package name as base-package
 Internally added : @ComponentScan("in.nareshit.raghu")

*) We can modify by adding same annotation at main class
     @ComponentScan("com.app.test")
  But it is not recomanded in realtime.

*) We can even provide multiple basePackages
  [it is also not recomanded]

At main class:
	@ComponentScan({
		"com.app.test",
		"in.nareshit.raghu",
	})


--FAQs----------------------------------------------------
Q) How many setup files are generated by Spring Boot Application?
A) 3 files
  > Starter class / Main class
  > Input files (properties/YAML)
  > Build File (pom.xml/build.gradle)

Q) What is the work of Spring Container?
A)
  > Find/Scan Your class, create object
  > inject data to variable
  > link objects
  > Destory the container

Q) Which annotation is used to create object to our class?
A)
   @Component

Q) How can change basePackage from main class package
	to some other package in Spring Boot?
A) 
   It is not recomanded.
   If you want to change basePackage, at main class

   @ComponentScan("com.app.test")

--Example-------------------
package com.app.test;

import org.springframework.stereotype.Component;

@Component
public class Employee {

}
--Main class---
package in.nareshit.raghu;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.ComponentScan;

@SpringBootApplication
@ComponentScan("com.app.test")
public class SpringBoot2BasePackTestApplication {
	//ctrl+F11
	public static void main(String[] args) {
		ApplicationContext ac = SpringApplication.run(SpringBoot2BasePackTestApplication.class, args);
		Object ob = ac.getBean("employee");
		System.out.println(ob);
	}

}
--------------------------------------------
Q) Can Provide basePackages  in Spring Boot?
A) Yes. Not recomanded.
	@ComponentScan({
		"com.app.test",
		"in.nareshit.raghu",
	})

Q) Who will provide @ComponentScan in Spring Boot?
A)
  By default this is given by @SpringBootApplication annotation.
  If programmer wants we can override.
